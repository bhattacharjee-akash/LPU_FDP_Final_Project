# Backend API Documentation
> **LPU Academic Copilot — REST API Specifications and Authentication Rules**

The FastAPI backend exposes endpoints to handle authentication sync, faculty profiles, settings, file uploads, log tracking, and PDF reports.

---

## 1. Authentication Requirements
Except for liveness check endpoints, all API endpoints expect an HTTP Bearer Token:
- **Header**: `Authorization: Bearer <SUPABASE_JWT_ACCESS_TOKEN>`

---

## 2. API Endpoints Reference

### **A. System & Diagnostics**

#### `GET /api/debug-logs`
* **Access**: Public
* **Description**: Returns the latest 50 logs and 10 histories from the database for status audits.
* **Response**:
  ```json
  {
    "logs": [
      {
        "id": 1,
        "syllabus_id": 12,
        "agent_name": "PlanningAgent",
        "status": "COMPLETED",
        "log_message": "Extracted syllabus details successfully.",
        "created_at": "2026-07-06T04:07:38"
      }
    ],
    "histories": []
  }
  ```

---

### **B. Faculty Profile & Settings**

#### `GET /api/profile`
* **Access**: Authenticated
* **Description**: Retrieve the current faculty profile (Name & Department).

#### `POST /api/profile`
* **Access**: Authenticated
* **Payload**:
  ```json
  {
    "name": "Dr. Amanpreet Singh",
    "department": "Computer Science & Engineering"
  }
  ```

#### `GET /api/settings`
* **Access**: Authenticated
* **Description**: Retrieve the user's AI configurations.

#### `PUT /api/settings`
* **Access**: Authenticated
* **Payload**:
  ```json
  {
    "llm_provider": "gemini",
    "model_name": "gemini-1.5-flash",
    "temperature": 0.7
  }
  ```

---

### **C. Syllabus Upload & Processing**

#### `POST /api/upload`
* **Access**: Authenticated
* **Content-Type**: `multipart/form-data`
* **Payload**: `file` (Binary PDF File)
* **Response**:
  ```json
  {
    "syllabus_id": 4,
    "filename": "machine_learning.pdf",
    "status": "PROCESSING",
    "message": "Syllabus parsing done. Agent workflow running in the background."
  }
  ```

#### `GET /api/status/{syllabus_id}`
* **Access**: Authenticated
* **Description**: Poll the multi-agent pipeline status and logs for a specific syllabus.
* **Response**:
  ```json
  {
    "syllabus_id": 4,
    "status": "PROCESSING",
    "logs": [
      {
        "id": 8,
        "syllabus_id": 4,
        "agent_name": "System",
        "status": "STARTED",
        "log_message": "Starting multi-agent orchestration pipeline.",
        "created_at": "2026-07-06T04:07:36"
      }
    ]
  }
  ```

---

### **D. Reports & Downloads**

#### `GET /api/report/{syllabus_id}`
* **Access**: Authenticated
* **Description**: Retrieve the full raw JSON data generated by all agents (lesson plan, assignments, quizzes, etc.).

#### `GET /api/download/{syllabus_id}`
* **Access**: Public *(Enables browser downloads without header conflicts)*
* **Description**: Compiles all database records for the syllabus and returns the generated report PDF file.
* **Response**: `application/pdf` binary stream.
